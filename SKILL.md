# Emil's Voice

Портативный движок голоса. Делает так, что текст звучит как Эмиль: его тон, ритм, лексика и структурные привычки — выжатые из реального корпуса его публичных постов и стримов.

## Применение (рантайм)

Когда нужно написать или переписать текст голосом Эмиля:

1. Прочитай `voice-profile.md` — это дистиллированный стиль-гайд (публичный голос: посты, стримы, лонгриды).
2. Если задача — диалоговый/чат-формат (ответ в переписке, постановка задачи, фидбэк), дополнительно прочитай `chat-style.md`.
3. Прочитай 2–3 файла из `examples/` — эталонные реальные посты, держи их перед глазами как образец ритма и тона.
4. Пиши/переписывай текст, применяя профиль. **API при этом не вызывается** — всё уже в файлах.
5. Перед выдачей прогони по чек-листу самопроверки (в конце `voice-profile.md`).

Язык голоса — **только русский**.

## Сборка и обновление корпуса

Запускается один раз при создании скилла и потом по мере появления нового контента.

```bash
# 1. Ключ ScrapeCreators (в .env или env-переменной)
export SCRAPECREATORS_API_KEY=...

# 2. Проверка ключа и формата ответа
node tools/collect.mjs --probe

# 3. Полный сбор корпуса → corpus/corpus.md
node tools/collect.mjs
```

Источники и хэндлы заданы в `config.json` (Threads `latypov.emil`, LinkedIn `rrotatew`, два YouTube-стрима).

## Построение voice-profile.md из корпуса

После сбора корпуса (`corpus/corpus.md`) построй профиль:

1. Прочитай весь `corpus/corpus.md`.
2. Выяви и опиши в `voice-profile.md`:
   - **Тон и регистр** — насколько прямой/ироничный/тёплый, дистанция с читателем.
   - **Ритм** — типичная длина предложений, короткие рубленые vs длинные, абзацы.
   - **Лексика** — слова-маркеры, любимые обороты, сленг, что повторяется.
   - **Структурные привычки** — как открывает (хук), как закрывает (CTA/вывод), как делает списки и акценты.
   - **Юмор и приёмы** — самоирония, метафоры, риторические вопросы.
   - **Анти-паттерны** — чего Эмиль НЕ делает (канцелярит, водянистые преамбулы, «как ИИ»).
3. Отбери 5–10 самых характерных реальных постов в `examples/` (по файлу на пост, имя `NN-threads.md` / `NN-linkedin.md`).
4. Допиши в конец `voice-profile.md` чек-лист самопроверки.

## Проверка качества (holdout)

Отложи 2–3 реальных поста, не показывай их при построении профиля. Сгенерируй текст на ту же тему по профилю и сравни стилистическое попадание с отложенными. Если профиль промахивается — уточни его.

## Структура

```
SKILL.md            ← этот файл
config.json         ← хэндлы и источники
voice-profile.md    ← публичный голос: регистры, маркеры, анти-паттерны (грузится всегда)
chat-style.md       ← чат-стиль: задачи, фидбэк, переписка (грузится по необходимости)
examples/           ← эталонные реальные посты
tools/collect.mjs   ← сборщик корпуса через ScrapeCreators API
corpus/             ← сырьё (в git не коммитится, регенерируется)
```
