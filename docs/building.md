# Сборка и обновление скилла

Нужно только при первичной установке и когда появился новый контент. В рантайме голоса ничего из этого не запускается.

## 1. Сбор корпуса

```bash
# Ключ ScrapeCreators (в .env или env-переменной)
export SCRAPECREATORS_API_KEY=...

# Проверка ключа и формата ответа
node tools/collect.mjs --probe

# Полный сбор корпуса → corpus/corpus.md
node tools/collect.mjs
```

Источники и хэндлы заданы в `config.json` (Threads `latypov.emil`, LinkedIn `rrotatew`, YouTube-стримы). Сырьё в git не коммитится, регенерируется.

## 2. Построение voice-profile.md из корпуса

После сбора (`corpus/corpus.md`):

1. Прочитай весь `corpus/corpus.md`.
2. Опиши в `voice-profile.md`:
   - **Тон и регистр** — прямой/ироничный/тёплый, дистанция с читателем.
   - **Ритм** — длина предложений, короткие рубленые vs длинные, абзацы.
   - **Лексика** — слова-маркеры, любимые обороты, сленг, что повторяется.
   - **Структурные привычки** — как открывает (хук), как закрывает (CTA/вывод), списки и акценты.
   - **Юмор и приёмы** — самоирония, метафоры, риторические вопросы.
   - **Универсальные правила** — что писать «как надо» во всех регистрах (позитивной формулировкой, каждое правило один раз).
3. Отбери 5–10 самых характерных реальных постов в `examples/` (по файлу на пост, имя `NN-threads.md` / `NN-linkedin.md`).
4. Держи `voice-profile.md` без changelog и без дублей: каждое правило живёт в одном месте, историю правок оставляй в git.
5. Допиши в конец `voice-profile.md` чек-лист самопроверки.

## 3. Проверка качества (holdout)

Отложи 2–3 реальных поста, не показывай их при построении профиля. Сгенерируй текст на ту же тему по профилю и сравни стилистическое попадание с отложенными. Если профиль промахивается — уточни его.
